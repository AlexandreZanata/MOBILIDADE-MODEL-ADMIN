# GovMobile Admin — Design System: Tokens & Styling

> **Philosophy:** Token-driven and accessibility-first. Every visual decision serves operational clarity.  
> **Stack:** Tailwind CSS v4 · Lucide React · i18next

---

## Five Design System Principles

1. **Operational clarity above aesthetic novelty** — operators work under pressure
2. **Token-driven** — no hardcoded values in any component
3. **Accessibility as a primary requirement**, not a post-launch concern
4. **Role-aware** — permissions via `<Can />`, never via `user.role === "ADMIN"`
5. **Predictable** — same appearance = same behavior, always

## Using Tokens in Code

```plaintext
// ✅ Preferred — Tailwind classes generated by @theme
<div className="bg-brand-primary text-white rounded-md">Primary action</div>

// ✅ For dynamic TypeScript values
import { tokens } from "@/theme/tokens";
const style = { color: tokens.colors.semantic.danger };
<span style={style}>Danger text</span>

// ❌ Forbidden — hardcoded color
<div style={{ backgroundColor: "#1a56db" }}>Bad hardcoded color</div>

// ❌ Forbidden — color outside tokens
<div className="bg-blue-600">Bad non-token class</div>

// ❌ Forbidden — hardcoded radius
<div style={{ borderRadius: "8px" }}>Bad hardcoded radius</div>
```

## Mandatory Style Rules

| Rule                                                     | Status    |
|----------------------------------------------------------|-----------|
| Use only Tailwind utility classes                        | Required  |
| Use design tokens for colors, radius, and fonts          | Required  |
| No hardcoded color values                                | Forbidden |
| No inline `style` (except documented dynamic values)     | Forbidden |
| No CSS Modules for component styling                     | Forbidden |
| All visible text via `useTranslation()`                  | Required  |
| No hardcoded role checks (`user.role === "ADMIN"`)       | Forbidden |
| Every exported component has a `data-testid` prop        | Required  |
| Every exported component has JSDoc                       | Required  |
| Every exported component has an explicit props interface | Required  |

## Toast Notifications

Managed by **Sonner** (`^2.0.7`). Configured in the root layout:

```tsx
<Toaster position="bottom-right" richColors />
```

Usage in mutation hooks:

```typescript
import { toast } from "sonner";

toast.success(t("toast.created"));
toast.error(t("toast.error"));
```
